# 示例：小红书试用扫描

## 用户输入

```text
我卖敏感肌防晒喷雾，主要卖给军训、通勤、户外拍摄的人。帮我先在小红书试搜一批客户。
```

## Skill 应该怎么做

1. 先读取 `get_workflow_manifest`。
2. 不要求 OPC 登录，不要求用户先建产品。
3. 建议 3 个搜索词：`军训防晒敏感肌`、`通勤防晒不闷痘`、`户外拍摄防晒补涂`。
4. 用户确认后调用 `search_keyword_for_leads`，传产品描述和 `platform=xiaohongshu`。
5. 汇报过程、客户名单、评论原话、跟进话术和解锁提示。

## 输出示例

```text
搜完了。我在小红书搜了「军训防晒敏感肌、通勤防晒不闷痘、户外拍摄防晒补涂」，实际看了 8 篇笔记，共读到 186 条公开评论，挑出 11 个潜在客户，其中 4 个高意向。

优先跟这几位：
1. 橙子要军训（高意向 · 购买咨询）：“下周军训，敏感肌，求不闷痘防晒。”
   跟进：可以从“敏感肌军训不闷痘”角度回复，给她使用场景和补涂建议。
2. momo（高意向 · 竞品不满）：“买过一个很油，下午直接闷闭口。”
   跟进：先共情油和闷闭口，再说明清爽和成分安全。

这批评论还显示，客户最关心的是“不闷痘、方便补涂、军训暴晒”。要不要我基于这些原话写一条小红书视频脚本？
```

## 为什么这个示例有效

- 它先试用扫描，不提前要求 OPC 登录。
- 它展示过程透明度：搜索词、内容数量、评论数量。
- 它不只给名单，还给了跟进话术和内容选题。
