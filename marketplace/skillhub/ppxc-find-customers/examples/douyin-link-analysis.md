# 示例：抖音链接评论分析

## 用户输入

```text
我做本地少儿体能课，这是一个抖音视频链接。帮我分析评论区里哪些家长可能想报名。
```

## Skill 应该怎么做

1. 先问清楚课程卖点，例如年龄段、城市、价格或试听课。
2. 用户给了具体链接时，跳过关键词搜索，直接调用 `analyze_video_comments`。
3. 如果抖音要求登录，只调用 `check_status_and_login(action=login_douyin)`，不要弹 OPC 登录。
4. 输出客户名单时区分高意向、可培育、内容选题。

## 输出示例

```text
分析完了。我读取了这个抖音视频下 94 条公开评论，筛出 7 个可能有报名需求的家长，其中 3 个高意向。

高意向：
1. 乐乐妈妈（高意向 · 价格咨询）：“这种课一周几次？孩子 5 岁能上吗？”
   跟进：先回答年龄和频次，再给试听课入口。
2. 城南小白（高意向 · 到店距离）：“在城南有校区吗？离我家近就想试试。”
   跟进：先确认位置，再给最近校区和试听时间。

可培育：
3. 小太阳（中意向 · 效果疑问）：“体能课对专注力有帮助吗？”
   跟进：适合用案例内容培育，不要直接催报名。

建议下一条内容拍“5 岁孩子第一次上体能课会做什么”，因为评论里反复出现年龄、频次和试听顾虑。
```

## 为什么这个示例有效

- 它把“家长可能报名”转成了可执行分类。
- 它避免承诺成交，只给优先级和跟进动作。
- 它从线索反推内容选题，体现增值能力。
